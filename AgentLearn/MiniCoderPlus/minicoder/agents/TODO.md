# Agent 角色演进规划 (TODO)

本文件夹旨在存放不同职责的自主代理。未来我们将把通用的 `agent.py` 逻辑抽象为基类，并派生出以下专门角色：

## 1. Coder (编码专家)
- **职责**：实现核心功能、编写算法、重构代码。
- **关联工具**：文件读写、Bash 执行、Git 操作。
- **状态**：当前 `agent.py` 的核心实现。

## 2. Reviewer (校对/审计员)
- **职责**：检查 Coder 产出的代码安全性、规范性及逻辑错误。
- **关联工具**：只读文件访问、Linter 接口。
- **特点**：拥有更严格的 System Prompt，专注于“挑错”而非“实现”。

## 3. Distributor (分发/部署员)
- **职责**：将 WorkSpace 内验证通过的代码分发到下位机或生产环境。
- **关联工具**：Embedded SSH (Base64 Pipe), SCP。
- **特点**：处理网络抖动、环境探测及远程验证。

## 4. Architect (架构师)
- **职责**：负责项目结构的整体规划，将大任务拆解为小任务分发给 Coder。
- **特点**：具备更强的多轮对话调度能力和全局上下文管理。

---
> *注：每个 Agent 应当能够独立搭载不同的 Tools 集合和 Prompt 模板。*
