# FileStation

FileStation 是一个极简、高性能且面向未来的现代文件管理系统。它不仅是一个开箱即用的 Web 网盘，更是一个可以作为**文件管理器基座**的统一抽象层，为后续对接 NAS (Network Attached Storage) 或私有云存储提供标准化的架构支撑。

它专注于提供稳健的文件存储底座，并为后续的 多模态索引及自动化标注预留了无限的插件化扩展能力。

##  核心理念

**Simple, Fast, and AI-Ready.**
FileStation 被设计为你的个人“文件数据中心”。我们剔除了所有复杂的概念，只保留最核心、最直观的文件操作体验，并使其具备极强的可嵌入性和扩展性。

##  主要功能

-  **底座级文件管理**：支持秒级的上传、下载、删除及跨文件夹拖拽移动，可作为独立组件嵌入其他系统。
-  **NAS 友好架构**：高度抽象的驱动系统，可轻松适配物理磁盘、S3 或其他网络存储协议。
-  **内容寻址存储**：底层支持 SHA256 内容指纹技术，天然支持文件去重（秒传）。
-  **多模态展示**：支持网格视图、列表视图及紧凑模式，适应不同管理习惯。
-  **插件化扩展 (Roadmap)**：
    - **MetaParser Plugin**：自动提取 Office、PDF 及图片的元数据，实现自动分类。

##  技术架构

本项目坚持**极简主义**架构逻辑，确保系统轻量且易于部署。

| 模块 | 技术选型 | 优势 |
| :--- | :--- | :--- |
| **后端 (Backend)** | FastAPI | 异步高性能，极速构建 REST 接口。 |
| **前端 (Frontend)** | React + TailwindCSS | 响应式设计，极致的桌面级 Web 交互。 |
| **数据库** | SQLite | 零配置，单文件持久化。 |
| **存储策略** | Plugable Driver System | 支持 Path (路径) 和 CAS (内容寻址) 两种模式，可插拔。 |

##  架构设计：可配置存储与 DB 控制

为了满足不同场景下的需求（如：开发者需要极致透明的路径管理，或者企业需要海量去重），FileStation 引入了**配置层中间件**封装。

### 1. 配置项 (`backend/app/core/config.py`)
- `STORAGE_MODE`: 可选 `"path"` 或 `"cas"`。
- `USE_DATABASE`: 布尔值。设为 `false` 时，系统将退化为纯物理磁盘扫描模式。

### 2. 存储驱动设计
系统通过 `StorageDriver` 接口实现了物理操作的解耦：
- **PathDriver**: 物理文件直接对应 UI 路径，删除和移动操作实时作用于磁盘。
- **CASDriver (Planned)**: 基于 Hash 的去重存储模式。

### 3. DB 是可选项
当 `USE_DATABASE` 为 `false` 时，`manager.py` 会跳过数据库操作，直接由驱动程序通过磁盘扫描（`os.scandir`）返回内容。

##  存储架构决策记录 (Decision Records)

在项目演进过程中，我们针对存储方案进行了深度讨论：

### 方案 A：内容寻址存储 (Content-Addressed Storage, CAS)
- **原理**：文件以 `SHA256` 乱序命名存储，数据库记录路径映射。
- **优势**：天然去重、极速物理移动、底层安全。
- **选择结论**：作为后续“高性能去重插件”的可选实现。

### 方案 B：传统路径存储 (Traditional Path Storage, TPS) - **当前采用**
- **原理**：`uploads/` 目录结构与用户在 UI 中看到的虚拟路径完全同步。
- **优势**：
    - **可见性**：即使用户关闭 FileStation，也能直接通过系统资源管理器管理文件。
    - **兼容性**：第三方 AI 工具、同步软件、文本编辑器可直接物理访问文件。
    - **可维护性**：数据库损坏时具备极强的零成本灾难恢复能力。
- **选择结论**：为了 FileStation 更好的通用性和 AI 兼容性，我们选择此方案作为底座标准。

##  快速开始

### 后端启动
```bash
cd backend
pip install -r requirements.txt
python run.py
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```
